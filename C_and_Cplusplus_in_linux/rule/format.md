# 编码约定

* 适当地在语句中增加空格
	* 关键字（if、for等）后面
	* 二元操作符两边
	    * 赋值操作符、比较操作符、算术操作符、逻辑操作符、位域操作符，如"="、"+=" ">="、"<="、"+"、"\*"、"%"、"/"、"&&"、"||"、"<<","^"等二元操作符的前后应当加空格
	    * "[]"、"."、"->"这类操作符前后不加空格
	* 逗号和不在行末尾的分号后面 
	* 逗号、分号和右括号的前面，左括号后面不跟空格；一元操作符不加空格
* 使用tab对齐
* 每行最多字符数不超过100个，包括tab所占字符数
* 注释只出现在有必要的地方，且要清晰简洁，不引起误解，注释采用/* */的形式
* 以开源代码为基础，对函数体、变量、数据类型等的修改，须在注释中加以说明
* 函数的注释只需说明函数的功能及必要的注意事项，无需说明具体的实现原理和方式
* 凡修改自其他开源软件的函数，均须在函数定义之前，通过注释对原函数的来源作出说明
* 所有的"{"都跟在语句后面
